# 05 自举计划

## 什么是自举

自举是指：一门语言的编译器最终由这门语言自己实现，并且能够编译自身。

本项目的目标是：

```text
C++ / Rust / Python 写的 hancc0
    -> 编译中文写的 hancc 源码
    -> 得到 hancc1
    -> hancc1 再编译同一份源码
    -> 得到 hancc2
    -> hancc2 继续编译自己
```

当 `hancc1`、`hancc2`、`hancc3` 行为一致时，说明自举基本成功。

## 阶段 0：宿主编译器

先用熟悉的语言写第一个编译器：

```text
hancc0
```

推荐实现语言：

```text
C++：贴近系统编程，方便后续移植。
Rust：类型安全，工程体验好。
Python：最快验证前端，但不适合长期作为主实现。
```

第一版建议：C++ 或 Rust。

## 阶段 1：最小语言

语言支持：

```text
整型
函数
局部变量
if
while
return
函数调用
四则运算
比较运算
```

能编译：

```han
整 主() {
    返 7;
}
```

## 阶段 2：写小程序

用 `.han` 写：

```text
阶乘
斐波那契
最大值
求和
冒泡排序
```

验证语言够用。

## 阶段 3：补齐编译器所需能力

为了用 `.han` 写编译器，需要加入：

```text
字符串
动态数组
结构体
枚举
指针
文件读取
文件写入
内存分配
命令行参数
哈希表或符号表
```

## 阶段 4：用中文语言重写部分编译器

先不要一次性重写完整编译器。

推荐顺序：

```text
词法.han
Token.han
动态数组.han
字符串.han
语法表达式.han
AST.han
类型.han
IR.han
后端_riscv.han
主程序.han
```

每重写一个模块，都用宿主编译器编译并测试。

## 阶段 5：完整自举

执行：

```bash
hancc0 编译器/主程序.han -o hancc1
hancc1 编译器/主程序.han -o hancc2
hancc2 编译器/主程序.han -o hancc3
```

检查：

```text
hancc1 能编译示例程序
hancc2 能编译示例程序
hancc2 能重新编译编译器
hancc2 和 hancc3 输出行为一致
```

## 阶段 6：自举后继续中文化

自举完成后再逐步做：

```text
中文错误信息
中文 IR 工具
中文汇编打印器
中文字节码 VM
中文标准库
RISC-V 裸机运行时
```

## 关键原则

```text
不要一开始追求完整 C。
不要一开始追求中文机器码。
不要一开始追求自然语言。
先让一个小语言能编译、能运行、能扩展。
```
