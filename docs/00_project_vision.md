# 00 项目愿景

## 一句话目标

用中文重新实现一套从 C-like 系统语言、编译器、IR、汇编到 RISC-V 后端与自举的工程化体系。

## 项目定位

`han-riscv` 不是中文宏替换项目，也不是文言文玩具语言项目。

它的核心目标是：

```text
中文源码 -> 自己的编译器前端 -> 自己的 IR -> RISC-V 后端 -> 自举
```

也就是说，本项目关心的是完整编译器工程，而不是简单把中文替换成英文再交给 C 编译器。

## 为什么先抄 C

C 是最适合作为第一阶段模板的语言：

1. 语义接近硬件。
2. 类型系统不复杂。
3. 函数、指针、结构体、数组足够表达编译器自身。
4. 能自然落到 RISC-V 这类真实 ISA。
5. 适合后续自举。

第一阶段不是发明全新语言，而是复制 C 的核心能力：

```text
函数
局部变量
整型
字符
指针
数组
结构体
if / while / for
return
基本运算
函数调用
```

然后逐步中文化：

```text
关键字中文化
错误信息中文化
标准库中文化
IR 中文化
汇编助记符中文化
字节码表示中文化
```

## 对“中文机器码”的边界定义

真实 CPU 执行的是二进制编码，不执行中文字符。

所以本项目中的“中文机器码”不是说物理 CPU 直接识别汉字，而是分层实现：

1. 中文源码：程序员写的语言是中文关键字。
2. 中文 IR：中间表示可以用中文打印。
3. 中文汇编：RISC-V 助记符可以有中文别名。
4. 中文字节码：可以设计汉字表示的 VM 指令。
5. 真实机器码：最终仍然编码成 RISC-V 二进制。

如果未来进入 FPGA / 自研 CPU，可以继续探索硬件层面的中文指令集展示，但那属于后续体系结构项目。

## 长期成果形态

最终希望形成：

```text
hancc          中文 C-like 编译器
han-as         中文汇编器
han-ir         中文 IR 工具
han-rt         中文运行时
han-vm         中文字节码虚拟机
hanlib         中文标准库
```

以及一份中文写成的自举编译器源码：

```text
编译器/
  词法.han
  语法.han
  语义.han
  类型.han
  中间码.han
  后端_栈机.han
  后端_riscv.han
  主程序.han
```
